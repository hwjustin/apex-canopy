# Contributing

Thanks for your interest in apex-canopy.

## Workflow

1. Fork the repo (or, if you have write access, create a branch — never push directly to `main`).
2. Create a topic branch off `main`:
   ```bash
   git checkout -b feat/your-thing
   ```
3. Commit your changes. Keep commits focused and messages descriptive.
4. Open a pull request against `main`.
5. A maintainer will review and merge. CI must be green.

## Issues

Bug reports, feature requests, and ideas are all welcome — open an [issue](https://github.com/hwjustin/apex-canopy/issues). Include:

- What you expected to happen
- What actually happened
- Steps to reproduce (for bugs)

## Code style

Run `pnpm build` before opening a PR to make sure types and the production build still pass.
